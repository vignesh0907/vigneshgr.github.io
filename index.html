Projects

I. Youtube Data Harvesting and Data Warehousing

Problem Statement:

The task is to build a Streamlit app that allows users to access and analyze data from multiple YouTube channels. Users can input a YouTube channel ID to access data like channel information, video details, and user engagement. The app should facilitate storing the data in a MongoDB database and allow users to collect data from up to 10 different channels. Additionally, it should offer the capability to migrate selected channel data from the data lake to a SQL database for further analysis. The app should enable searching and retrieval of data from the SQL database, including advanced options like joining tables for comprehensive channel information.

Solution:

  • Start by setting up a Streamlit application using the python library "streamlit", which provides an easy-to-use interface for users to enter a YouTube channel ID, view channel details, and select channels to migrate. And few given questions to view
  • Establish a connection to the YouTube API V3, which allows me to retrieve channel and video data by utilizing the Google API client library for Python.
  • Store the retrieved data in a MongoDB data lake, as MongoDB is a suitable choice for handling unstructured and semi-structured data. This is done by a method to retrieve the previously called api and storing the data in the database collections.
  • Transferring the collected data from multiple channels namely the channels,videos and comments to a SQL data warehouse, utilizing a SQL database like MySQL or PostgreSQL for this purpose.
  • Utilize SQL queries to join tables within the SQL data warehouse and retrieve specific channel data based on user input. For that the SQL table previously made has to be properly given the the foreign and the primary key.
  • The retrieved data is displayed within the Streamlit application

Skills Gained On:

  • Applications: Python, MongoDB, MySQL, Git
  • IDE: Visual Studio Code
  • Libraries: Streamlit, Pandas, RegEx, SQLalchemy

IV. Airbnb Analysis

Problem Statement:

This project aims to analyze Airbnb data using MongoDB Atlas, perform data cleaning and preparation, develop interactive geospatial visualizations, and create dynamic plots to gain insights into pricing variations, availability patterns, and location-based trends.

Solution:

  • Establish a Connection between Streamlit Application and Mongo DB Atlas to retrive the Airbnb Dataset.
  • After retriving the data from the Mongo DB Atlas need to perform Data Cleaning, Address Missing Values, Handling Duplicates and conversion of Data Types using Pandas.
  • Need to create visualizations like interactive maps showcasing the distribution of Airbnb listings and other relevant factors like Prices, Ratings, availability patterns across seasons, occupancy rates, demand fluctuations, etc.
  • With the help of Descriptive Analysis we conduct price analysis and exploring it's variation based on location, property type and seasons and visualizing those using dynamic plots and charts.
  • Atlast built an Dashboard using Tableau with filters and various visualizations to present key insights from the analysis.

Skills Gained On:

  • Applications: Python, MongoDB, Data Preprocessing, Visualization, EDA, Tableau, Git
  • IDE: Visual Studio Code
